# NetDash 릴리스 노트

## v3.6.1 (2026-06-30) — 자격증명 복호화 버그 수정

- 증상: 저장된 API 토큰으로 수집 시 "token 또는 username/password 중 하나 필요" 오류.
- 원인: DPAPI 복호화에서 `CryptUnprotectData` 반환값의 잘못된 인덱스(`[0]` 빈 설명)를
  사용해 복호화가 항상 실패했습니다(저장된 토큰이 빈 값으로 전달됨).
- 수정: 데이터 인덱스(`[1]`)를 사용하도록 고쳐 저장된 토큰으로 정상 수집됩니다.

## v3.6.0 (2026-06-30) — FortiGate ARP 404 수정 + 자격증명 재사용

**FortiGate ARP 수집 404 수정**
- 증상: 방화벽 수집 시 `404 ... /api/v2/monitor/router/arp` 오류.
- 원인: FortiOS 버전에 따라 ARP REST 경로가 다른데(7.x는 `network/arp`), 한 경로만
  사용해 해당 장비에서 404가 났습니다.
- 수정: ARP 경로를 **버전별로 순서 시도**(`network/arp` → `router/arp`)하고, 어떤
  경로도 없으면 예외 없이 빈 결과로 처리해 수집 전체가 실패하지 않게 했습니다.

**자격증명 재사용 (매번 토큰 입력 불필요)**
- 방화벽 등록 또는 첫 수집 때 입력한 API 토큰/계정을 **암호화 저장**합니다.
- 이후 수집 버튼을 누르면 **저장된 자격증명으로 바로 수집**(입력창 생략)합니다.
- FortiGate는 **API 토큰만으로 통신**됩니다 — 토큰만 넣으면 아이디/비밀번호 불필요.
- (저장 자격증명은 암호화되며 화면·API로 노출되지 않습니다)

## v3.5.3 (2026-06-30) — 연결 테스트 출발지 IP 표시/안내

- 연결 테스트 결과에 **실제 사용한 출발지(접근) IP**를 함께 표시합니다.
- 헤더 "접근 IP"가 **"자동(OS 기본)"**이면, OS가 임의 IP(127.0.0.1 등)를 출발지로
  고를 수 있다는 안내를 띄워 이더넷 IP를 선택하도록 유도합니다.
- (연결 테스트는 기존에도 설정한 접근 IP로 바인딩했으나 화면에 보이지 않아 확인이
  어려웠던 점을 개선)

## v3.5.2 (2026-06-29) — 버그 검증 후속 수정

전체 코드 버그 검증(Opus 2종 + 199 테스트) 결과 발견한 항목 수정:
- **UI 레이아웃/색상 깨짐 수정**: CSP가 inline `style`까지 차단하던 것을 `style-src
  'unsafe-inline'`로 허용(스크립트는 'self' 유지 → 보안 동일).
- **수정 시 중복 오류 메시지 개선**: 이름/IP(스위치)·호스트(방화벽)가 이미 존재하면
  불명확한 500 대신 "이미 사용 중" 안내(409).
- **Extreme(EXOS) 수집 개선**: 페이징 비활성화에 `terminal length 0`(Cisco용) 대신
  벤더별 명령(`disable clpaging`)을 사용하고, 실패해도 수집을 계속합니다.

## v3.5.1 (2026-06-29) — 수집 read_timeout 오류 + 수정/삭제 버튼 무반응 수정

- **수집 `read_timeout` 오류 수정**: `read_timeout`은 netmiko `send_command()`의 인자인데
  `ConnectHandler` 생성자에 잘못 전달해 `unexpected keyword argument 'read_timeout'`
  오류가 났습니다. send_command로 옮겼습니다.
- **수정/삭제 버튼 무반응 수정**: 보안 정책(CSP)이 inline onclick을 차단해 동적 버튼
  (수정·삭제·상세·수집)이 동작하지 않았습니다. 이벤트 위임 방식으로 전환해 CSP를
  유지하면서 정상 동작합니다.

## v3.5.0 (2026-06-29) — 수집 벤더 오류 수정 + 수정 기능

**스위치 수집 'unsupported device_type' 오류 수정**
- 증상: 연결 테스트는 성공하나 정보 수집 시 `unsupported 'device_type'` 오류.
- 원인: 수집 경로가 벤더 값(`cisco`)을 netmiko 타입으로 그대로 사용(연결 테스트는 매핑함).
- 수정: 수집 경로도 `cisco→cisco_ios` 등으로 정규화(device_type·명령·파서 일관).

**스위치 / 방화벽 수정 기능 추가**
- 스위치 현황 표·방화벽 목록의 **"수정"** 버튼으로 등록 정보(이름/IP/벤더/포트 등)를 변경.
  (방화벽 수정 시 자격증명은 유지되며, 변경하려면 재등록)
- 삭제 기능(기존)과 함께 잘못 등록한 장비를 정리할 수 있습니다.

## v3.4.2 (2026-06-29) — 수집 재시도/출발지 바인딩 안정화

- 스위치 수집(SSH)의 재시도 로직이 2회차부터 내부 설정이 비워져 깨지던 버그를 수정했습니다.
  이제 재시도 시에도 출발지 IP 바인딩(접근 IP 설정)이 일관되게 적용됩니다.
- 1회차 연결 실패(일시 장애 등) 후 재시도가 정상 동작합니다.

## v3.4.1 (2026-06-29) — 핫픽스 (netmiko 번들 누락)

- 증상: 스위치 연결 테스트/수집 시 `No module named 'netmiko'`.
- 원인: SSH 라이브러리(netmiko)가 함수 내부 lazy import라 PyInstaller가 일부만 번들,

- 수정: 빌드 설정에서 `collect_all`로 netmiko/paramiko **전체(모듈+데이터+바이너리)**를
  번들 → 스위치/Palo Alto SSH 및 연결 테스트가 정상 동작.

## v3.4.0 (2026-06-29) — M12 출발지 IP 바인딩 (장비 ACL 통과)

**장비 접근 출발지(source) IP 선택**
- PC에 NIC가 여러 개(물리/VPN/가상)일 때, 장비로 나가는 출발지 IP가 장비 ACL 허용 IP와
  달라 차단되던 문제를 해결합니다.
- 헤더의 **"접근 IP"** 드롭다운에서 출발지로 쓸 PC 이더넷 IP를 선택하면, 이후 모든 장비
  접근(스위치 SSH, FortiGate REST, Palo Alto SSH, 연결 테스트)이 그 IP로 바인딩됩니다.
- "자동(기본 라우팅)" 선택 시 기존처럼 OS가 결정합니다.
- 선택값은 PC의 실제 이더넷 IP 목록으로 제한(오설정 방지).

## v3.3.1 (2026-06-29) — 스위치 등록 오류 수정 + 삭제 기능

**스위치 수동 추가 DB 오류 수정**
- 증상: 스위치 등록 시 `DB_ERROR` / `ADD_SWITCH_MANUAL_ERROR` (구버전 스키마 DB).
- 원인: 등록 로직이 구버전 `switches`에 없는 컬럼(hostname/vendor/location/status/alert)을

- 수정: 실제 존재하는 컬럼만 골라 동적으로 INSERT/UPDATE → 어떤 스키마의 DB에서도 안전.

**삭제 기능 추가**
- 스위치 현황 표·방화벽 목록에 **삭제** 버튼 추가(잘못 등록 시 제거).
- 스위치 삭제 시 관련 수집 데이터(스냅샷/포트/MAC/ARP/이벤트)도 정리(호스트 인벤토리는 보존, 위치만 무효화).
- 방화벽 삭제 시 인터페이스/ARP도 함께 제거.

## v3.3.0 (2026-06-29) — M11 장비 연결성 / 등록 개선

**연결 테스트 (수집 전 선검증)**
- 스위치/방화벽 수집 모달에 "연결 테스트" 버튼 추가. IP+계정 입력 후 즉시 접근 가능
  여부를 확인(TCP 도달성 → 인증). 결과를 색상으로 표시.

**방화벽 API Key/자격증명 저장**
- 방화벽 추가 시 API 토큰/계정을 입력하면 **DPAPI로 암호화하여 저장**, 이후 수집 때
  재입력이 필요 없습니다. (자격증명 blob은 어떤 조회 API에도 노출되지 않음)
- 수집 결과(done/failed)가 목록에 상태로 표시됩니다.

**PC 이더넷 IP 표시**
- 헤더에 PC의 이더넷 IP를 표시합니다. 장비 접근에 사용되는 실제 IP를 안내하며,
  127.0.0.1(루프백)로는 장비에 접근할 수 없음을 알려줍니다.

**보안**: 연결 테스트/등록/수집 모두 host 허용대역·port(1-65535) 검증(SSRF 방어). 저장된
자격증명 blob은 GET 응답·UI에 노출하지 않음.

## v3.2.2 (2026-06-29) — 핫픽스 (ON CONFLICT 전면 제거)

v3.2.1은 스위치 수동 추가만 고쳤으나, 동일 원인이 **엑셀 업로드(호스트/장부 적재)와
방화벽 등록**에도 있었습니다. 구버전 스키마 DB에서 해당 UNIQUE 제약이 없으면 같은
`ON CONFLICT` 오류가 났습니다.

- 수정: `save_hosts`, `save_ledger_hosts`, `save_firewall`의 `ON CONFLICT`를 모두
  제거하고 수동 UPSERT(SELECT→UPDATE/INSERT)로 전환 — 어떤 스키마의 DB에서도 안전.
- 기존 보존 로직(측정/장부 컬럼 분리, 빈값 시 기존값 유지)은 그대로 유지.

## v3.2.1 (2026-06-29) — 핫픽스 (수동 추가 Internal server error)

**구버전 DB에서 스위치 수동 추가 실패 수정**
- 증상: 스위치 수동 추가 시 `Internal server error` —
  "ON CONFLICT clause does not match any primary key or unique constraint".
- 원인: 구버전 스키마로 생성된 DB의 `switches` 테이블에 `name` UNIQUE 제약이 없는데,
  `import_switches_bulk`가 `ON CONFLICT(name)`을 사용.
- 수정: `ON CONFLICT` 의존을 제거하고 name 기준 **수동 UPSERT(SELECT→UPDATE/INSERT)**로
  변경 — UNIQUE 제약 유무와 무관하게 모든 DB에서 안전하게 동작(멱등 유지).

## v3.2.0 (2026-06-29) — M10 방화벽 현황 (Palo Alto / Fortinet)

**방화벽 모니터링 기능 추가**
- 방화벽 장비(Fortinet FortiGate, Palo Alto PAN-OS)를 등록하고 **인터페이스 + ARP
  (연결된 IP/MAC)** 를 수집해 "방화벽 현황" 탭에 표시합니다.
- FortiGate: REST API(토큰/계정) 또는 SSH. Palo Alto: SSH CLI(netmiko).
- 방화벽 추가 모달 + 수집(자격증명 입력) 모달 + 장비별 인터페이스/ARP 상세 뷰.
- 자격증명은 수집 시점에만 사용하고 저장하지 않습니다.

**보안**
- 방화벽 host는 등록·수집 양 시점 모두 허용 대역(allowed_ip_ranges)으로 검증, port는
  정수 1-65535로 강제(SSRF 방어, CWE-918). FortiGate 무검증 TLS는 경고 로깅.

## v3.1.4 (2026-06-29) — 실행 투명성 + 로컬 인증/UI 수정

**보안·투명성 (콘솔 표시 전환)**
- 배경: 이전 빌드는 `console=False`(windowed)라 콘솔 없이 백그라운드에서 서버를 열고
  브라우저를 자동 실행했습니다. 이 패턴은 백도어/드로퍼와 구분되지 않아 EDR·SmartScreen이
  차단할 수 있습니다.
- 수정: **`console=True`로 전환**하여 실행 시 콘솔 창이 보이고, 접속 주소·실행 모드·종료
  방법을 안내합니다. 프로세스가 가시적이고 사용자가 직접 제어(창 닫기=종료)할 수 있습니다.

**로컬 인증 수정 (수동 추가 등 'unauthorized' 해결)**
- 증상: 스위치 수동 추가 등 모든 작업에서 `unauthorized` 오류.
- 원인: production 모드가 모든 `/api/*`에 토큰을 강제했으나 로컬 UI는 토큰을 보내지 않음.
- 수정: **루프백 바인딩 + 로컬(127.0.0.1) 요청은 인증 면제**(같은 머신 사용자만 접근).
  외부 노출 바인딩(0.0.0.0 등)일 때만 토큰을 강제(원격 접근 방어).

**UI**
- 수동 추가 벤더 목록에서 Juniper 제거.

M5~M8 네 개 마일스톤을 묶은 메이저 릴리스. 비동기 자격증명 보안 강화,
ExtremeXOS 파서 고도화, 그리고 장부 대조(Reconcile) 기능을 엔진부터 UI까지 완성.
(v2.0.0은 M4 릴리스가 점유하여 다음 메이저 v3.0.0으로 발행.)

### 주요 변경

**M5 — 비동기 자격증명 처리 (보안)**
- SSH 자격증명을 수집 큐 페이로드에서 제거하고 세션 저장소 경유로 전환 (CWE-522).
- 워커가 사용 시점에 로드하고 수집 종료(성공/실패/예외) 시 즉시 폐기.
- 세션 저장소 스레드 락 추가(CWE-362), 방어적 복사본 반환.

**M6 — ExtremeXOS 포트 표기 정규화 고도화**
- 잘못된 Cisco 'Gi' 접두사 제거, ExtremeXOS native 표기(slot:port / standalone) 보존.
- 가변 폭 Type 컬럼·들여쓰기·'Up-Link' 류 토큰에 견고한 토큰 기반 파싱.

**M7 — 장부 대조(Reconcile) 엔진**
- 엑셀 장부(기대 인벤토리) vs 실측(수집) 위치를 6판정으로 대조
  (일치/포트 불일치/스위치 불일치/장부에만/실측에만/정보없음).
- hosts 테이블 장부 컬럼 + 측정/장부 양방향 UPSERT 분리(적재 순서 무관 멱등).
- 비교용 포트 정규화(서브인터페이스 보존, 논리포트 충돌 방지, Juniper 커버).
- `GET /api/reconcile` 엔드포인트, 엑셀 장부 적재 경로(연결스위치/연결포트).

**M8 — 장부 대조 UI 패널**
- "장부 대조" 탭: 판정 요약 카드 + 호스트별 판정 테이블.
- 모든 동적 필드 XSS 이스케이프, 판정 배지 색상 구분.

### 품질
- 전체 테스트 106/106 통과 (회귀 0).
- 코드 리뷰: M5·M6 Codex, M7·M8 Claude Opus 4.8 (다중 라운드 PASS).
- Windows 단일 실행파일(netdash.exe) 빌드 + 스모크 테스트 통과.

### 배포
- `netdash.exe` (PyInstaller 단일 실행파일, console 비활성).
- 실행: 기본 `127.0.0.1:8082`. 데모 모드는 `DEMO_MODE=true` 환경변수.
- 프로덕션은 `API_TOKEN` 환경변수 필수.
