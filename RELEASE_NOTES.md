# NetDash 릴리스 노트

## v3.24.1 (2026-07-01) — ExtremeXOS 수집 실패 수정(실제 명령/형식)

- **ExtremeXOS 스위치 수집이 실패하던 문제 수정.** 원인은 `show ports`가 자동 갱신 화면이라
  응답이 끝나지 않아 멈춘 것 + MAC/ARP 명령이 EXOS 문법과 달랐던 것.
  - 포트 상태: `show ports no-refresh` (자동 갱신 방지) — 링크 상태 active/ready/disabled 인식.
  - MAC 테이블: `show fdb`, ARP: `show iparp`, 로그: `show log messages memory-buffer`.
  - 파서를 실제 EXOS 출력 형식(fdb/iparp/no-refresh)에 맞게 재작성.
- **수집 견고성 개선**: 한 명령이 미지원/타임아웃이어도 나머지는 수집하고, 원본 출력을
  저장하도록 명령별 개별 예외 처리(전부 실패 시에만 수집 실패 처리).
- EXOS 벤더 별칭 확대(extremexos/exos/extreme_xos 등 → extreme_exos).

## v3.24.0 (2026-07-01) — 벤더 자동 학습(수집 시 show version)

- **벤더가 '알 수 없음(unknown)'인 스위치를 수집하면 `show version`으로 실제 벤더를
  자동 학습**해 DB에 반영하고 현황판/스위치 리스트에 표시합니다.
  - 지원 인식: Cisco IOS/IOS-XE, Cisco Nexus(NX-OS), Arista(EOS), Extreme(EXOS), Juniper(JUNOS).
  - 학습된 벤더의 명령/파서로 이번 수집부터 맞춰 처리하고, 다음 수집부터는 처음부터 올바른
    장비 타입으로 접속합니다.
- 엑셀 가져오기로 vendor 없이 등록된 장비(또는 방화벽 unknown)도, 수집만 하면 벤더가 채워집니다.

## v3.23.1 (2026-07-01) — 엑셀 가져오기 장비 등록 수정 + 포트채널 포맷

- **엑셀 가져오기로 등록한 장비가 현황판/스위치 리스트에 안 나오던 문제 수정.**
  - 예전엔 **'vendor' 열이 있어야만 스위치**로 등록되고, 없으면 전부 '호스트(장부)'로
    분류돼 화면에 안 보였습니다. 이제 **장비 식별 컬럼(name/subnet/location 등)만 있어도
    스위치로 등록**하고, IP만 있는 목록도 등록합니다.
  - **hostname으로 스위치/방화벽 자동 구분**: hostname에 **`FW`가 있으면 방화벽**,
    아니면 스위치로 등록합니다(예: `DMZ-FW-01` → 방화벽, `..._SW1` → 스위치).
    방화벽은 벤더 미지정으로 등록되니 방화벽 현황에서 벤더/계정을 설정하세요.
  - 임포트 결과 안내에 방화벽 개수 추가.
- **NX-OS 포트채널 표기 `Po10`/`Po10(SU)` 모두 인식**(플래그 괄호 유무 무관).

## v3.23.0 (2026-07-01) — NX-OS 포트채널 멤버 해석(설비/TPS 백본 직결)

- **백본(NX-OS)에 포트채널(Po)로 직결된 TPS/설비의 실제 물리 포트가 확인되지 않던 문제 수정.**
  - NX-OS 수집 시 `show port-channel summary`를 함께 읽어 **Po → 물리 멤버포트**를 저장.
  - 설비 대조에서 MAC이 Po로만 보이면 **멤버 물리포트로 해석**해 직접 연결로 표시.
    예: 백본 Po10 → **"Eth1/1, Eth1/2 (Po10)"** + 연결 스위치=백본, 직접 연결로 인식.
- 적용: **백본 NX-OS 스위치를 이 버전으로 다시 일반 수집**한 뒤, 설비 현황에서
  **"새로고침(재매칭)"** 을 누르면 즉시 반영됩니다.

## v3.22.0 (2026-07-01) — 현황판 위치 통합 · UI 정리 · 검색 강화 · 설비 추출

- **현황판 랙 뷰에서 백본 스위치·방화벽 위치가 "위치 미상"으로 나오던 문제 수정.**
  TPS 호스트네임이 아니어도 **위치(location)** 로 그룹핑(서버실 랙 A09U27 / 자유 위치 텍스트).
  방화벽도 현황판(카드·랙 뷰)에 함께 표시됩니다(🛡).
- **서버실 현황 카드 뷰에서 방화벽과 스위치 표현 통일.** 방화벽 카드를 스위치 카드와 동일한
  골격으로 렌더링(현황판·서버실 공용).
- **'스위치 현황' 탭 → '스위치 리스트'** 로 이름 변경.
- **'장부 대조' 탭 제거**(사용자 요청).
- **상단 검색 강화:** HOSTNAME·IP·MAC 어떤 값으로도 검색. 특히 **MAC은 형식 무관**
  (`0050.56a1.b2c3` / `00:50:...` / `0050-56a1-...` / 무구분 모두 매칭). 방화벽은 위치로도 검색.
- **설비 현황 전체를 엑셀(xlsx)·TXT로 추출** 버튼 추가(TXT는 엑셀에서 한글 정상 표시되는 UTF-8 BOM).

## v3.21.0 (2026-07-01) — 설비 백본 직접연결 인식 + 방화벽 인터페이스 IP/위치

- **백본(NX-OS) 등에 직접 연결된 설비가 "직접 연결 미확인"으로 나오던 문제 수정.**
  물리 포트로 관측된 위치가 하나뿐이면(다른 스위치는 Po/Vl 업링크로만 보임) 그 스위치를
  직접 연결로 인식합니다. 포트가 MAC을 많이 학습하는 백본/서버 포트도 정상 인식.
- **방화벽 인터페이스 IP가 실제와 다르게 나오던 문제 수정.**
  - FortiGate: 설정값(cmdb) 대신 **실제 런타임 IP(monitor 엔드포인트)** 우선 사용
    (DHCP/PPPoE 할당 IP 반영), 실패 시 cmdb로 폴백.
  - Palo Alto: `show interface logical`에서 **실제 인터페이스 IP** 파싱(기존엔 IP 비어 있었음).
- **방화벽 등록에 위치(location) 추가.** `A09U27` 형식으로 넣으면 **서버실 현황**에
  스위치와 함께 랙/카드 뷰로 표시됩니다(🛡 아이콘). 방화벽 목록에도 위치 열 표시.

## v3.20.2 (2026-07-01) — 구형 장비 SSH 접속 오류(incompatible ssh peer) 수정

- 계정이 정상인데도 **"연결 실패 [auth] ... incompatible ssh peer"** 로 접속이 안 되던 문제
  수정. 원인은 최신 paramiko(5.0)가 구형 스위치의 SSH 알고리즘(SHA1 기반 KEX,
  ssh-rsa 호스트키, aes-cbc)을 **구현에서 제거**해 협상이 실패한 것.
  - 레거시 구현이 남은 **paramiko 3.5.1로 고정**.
  - `core/ssh_compat.py`가 SHA1 KEX·ssh-rsa·aes-cbc 등을 **폴백 선호목록에 재추가**
    (강한 알고리즘이 여전히 우선 협상 — 보안 저하 최소화).
- 연결 테스트 오류 메시지를 알기 쉬운 한국어 안내로 개선(협상 실패/인증 실패/타임아웃 구분).

## v3.18.3 (2026-06-30) — 설비 직접연결 스위치 판별 정확도 개선

실장비 피드백 반영. 이전에는 설비 MAC이 여러 스위치에 보이면 **Po(포트채널)·Vl(VLAN/SVI)
같은 업링크 경유 포트까지 "연결 스위치"로 잘못 표시**했습니다. 이제 진짜 직접 연결을 가려냅니다.

- **논리 포트(Po·Port-channel·Vl·Vlan·Lo·Tu·ae 등)는 직접 연결에서 제외.** 그 MAC은
  업링크/트렁크를 타고 넘어온 것이므로 설비가 직접 붙은 곳이 아닙니다.
- **물리 포트(Gi·Te·Fa·Eth 등) 중 "해당 포트의 MAC 수가 가장 적은 포트 = 액세스 포트"**
  를 직접 연결로 선택합니다. 트렁크 포트는 MAC이 많아 자동 배제됩니다.
- 각 설비에 **직접 / 간접·업링크 배지**를 표시. 직접 연결을 확신할 수 없으면(논리 포트뿐이거나
  트렁크만 관측) "간접·업링크"로 정직하게 표기하고, 그 외 관측 위치를 함께 보여줍니다.
- (정확도를 높이려면: 대역의 **각 TPS 스위치를 일반 수집**해 최신 MAC 테이블을 채운 뒤
  11번에서 대역 수집을 하세요. 액세스 스위치가 미수집이면 트렁크로만 보여 "간접"으로 잡힙니다.)
- DB: facility_hosts에 `direct`·`via` 컬럼 추가(기존 DB 자동 마이그레이션).

## v3.18.2 (2026-06-30) — 상단 검색 보강(설비·MAC) + 설비 수집 크래시 수정

- **상단 검색이 설비 현황·MAC 테이블을 못 찾던 문제 수정.** 이전에는 등록
  스위치/방화벽·ARP(IP만)·장부만 검색해, 수집된 **설비 IP**나 **MAC**으로
  검색하면 0건이 나왔습니다. 이제 다음을 모두 검색합니다:
  - 설비 현황(facility_hosts) — **IP·MAC** 모두
  - 스위치 MAC 테이블(최신 스냅샷) — **MAC**으로 어느 스위치·포트인지
  - ARP(스위치·방화벽) — **IP뿐 아니라 MAC**으로도
  - 기존 등록 스위치/방화벽·장부 호스트(IP·이름)
- 검색창 안내문구를 "IP·이름·MAC 검색"으로 갱신.
- **설비 대역 수집 완료 시 크래시 수정**: 완료 직전 `len(hosts)`가 미정의 변수라
  `show ip arp` 후 저장 단계에서 예외가 발생하던 버그(NameError)를 바로잡음.

## v3.2.2 (2026-06-29) — 핫픽스 (ON CONFLICT 전면 제거)

v3.2.1은 스위치 수동 추가만 고쳤으나, 동일 원인이 **엑셀 업로드(호스트/장부 적재)와

## v3.2.1 (2026-06-29) — 핫픽스 (수동 추가 Internal server error)

  "ON CONFLICT clause does not match any primary key or unique constraint".
- 원인: 구버전 스키마로 생성된 DB의 `switches` 테이블에 `name` UNIQUE 제약이 없는데,
